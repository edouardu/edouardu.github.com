Markup

Text formatting is done using custom flavoured Markdown syntax.

Inline elements

  • *emphasis*
  • **bold**
  • ***bold emphasis***
  • --strikethrough--
  • __underline__
  • $$spoiler$$
  • `code`

Block elements

Lists

- one
- two
- three

Also works with * and + markers. Ordered lists are also supported.

Blockquotes

> Who the hell
am I quoting?

Note the the space after the >

Fenced code

~~~
function make_american($com) {
  if (stripos($com, "america")!==FALSE) return $com; //already american
 
  $com = rtrim($com);
  $end = '!';
 
  if ($com == "") return $com;

  if (preg_match("/([.!?])$/", $com, $matches)) {$end = $matches[1]; $com = substr($com, 0, -1);}
 
  $com .= " IN AMERICA".$end;
 
  return $com;
}
~~~

Also works with ``` markers. Syntax is highlighted depending on the board.

Empty lines separate block elements

To break out of a blockquote:

> This is a blockquote.
this is still the same blockquote.

This is no longer a blockquote but a new paragraph.

Same for lists and other block elements:

This is a **paragraph** with __some__ markup in it.
- Still inside the paragraph...
- This will not turn into a list

- But this
- will

...and post quoting, this doesn't work for example:

>>5
~~~
  This won't turn into a fenced code block
  because we are already in a block context
  started by the quote '>>>5'
~~~

this works:

>>5

~~~
  Adding an empty line exits the block context,
  this will be converted to a fenced code block
~~~
